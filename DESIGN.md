# 바른발음 디자인 핸드오프 브리핑

## 프로젝트 개요

**앱명:** 바른발음 — 아동 조음 홈케어 SaaS  
**성격:** 아이와 부모가 함께 발음 교정 연습을 하는 게임화된 앱  
**레이아웃:** 모바일 전용 (`max-w-lg`, 512px), 가운데 정렬

### 둥근 모서리 (마시멜로 테마)

- `sm`: 12px
- `md`: 20px
- `lg`: 28px
- `xl`: 36px
- `full`: 9999px

### 그림자

- **bubble:** `0 4px 20px rgba(180,120,80,0.10)`, `0 1px 4px rgba(180,120,80,0.08)`
- **hover:** `0 8px 32px rgba(180,120,80,0.15)`, `0 2px 8px rgba(180,120,80,0.10)`

### 타이포그래피

- **폰트:** Pretendard Variable
- **폴백:** Pretendard, -apple-system, BlinkMacSystemFont, "Apple SD Gothic Neo", "Noto Sans KR", sans-serif

- 섹션헤더: `font-bold` (700)
- 라벨: `font-semibold` (600)
- 글자 크기: 36px(히어로) → 24px → 20px → 16px → 14px → 12px

| 구역 | 내용 |
|------|------|

| 프라이싱 CTA | 단일 카드, 민트 or 라벤더 배경, 구독 버튼 |

### 2. 로그인 `/login`

**배경:** 그라디언트 동일

- 중앙 흰 BubbleCard 1개
- `🐣` 이모지 float 애니메이션

- 구글 버튼 (흰 배경, SVG 아이콘, 라이트 보더)

### 4. 대시보드 홈 `/dashboard`

**헤더:** 고정(sticky), 글래스모피즘 (`rgba(253,250,245,0.85)` + blur 12px)  
**하단 내비:** 5개 탭 (홈 / 오답노트 / 아이연습 / 저장 / 설정), 동일 글래스 처리

| 섹션 | 내용 |
|------|------|
| 인삿말 | 바른발음 로고(56px) + 아이 이름 |
| 통계 4열 | "학습 단어수 / 연습시간 / 연속출석 / 정확도" — 각 카드에 이모지 아이콘 |
| 최근 연습 기록 | 날짜 + 단어수 + 정확도 리스트 |
| 오답노트 CTA | 피치 버튼 "오답노트 작성하기" |

### 5. 오답노트 `/dashboard/note` — 부모 전용

**목적:** 부모가 아이의 오발음을 기록 → AI가 원인 분석 + 약점 분석 + 해결책 제시 → 아이연습 연결

**배경:** 아이보리

| 섹션 | 내용 |
|------|------|
| 입력 폼 | 옳은 발음(목표 단어) 입력 필드 + 아이의 발음(오답) 입력 필드, 피치 포커스 보더 |
| AI 분석하기 버튼 | 피치 bg, 하단 고정 |
| AI 분석 결과 카드 | 라벤더 bg BubbleCard — 오류 음소 표시(`ㅅ→ㄷ`), 원인 설명(쉬운 말), 핑크 배지 |
| 누적 약점 분석 카드 | 피치 라이트 bg — 최근 20개 오답 데이터 분석, "마찰음 'ㅅ'이 파열음 'ㄸ'으로 대치되었습니다" (전문 용어 + 쉬운 설명), 약점 음소 배지(핑크) 표시 |
| 원인 분석 섹션 | 흰 bg — "왜 '따과'라고 발음할까요?" 섹션, 조음 원리를 쉬운 말로 설명 (예: "혀가 입천장을 '쾅' 닫아서 강한 소리가 나요"), 혀 위치를 보여주는 간단한 다이어그램/애니메이션 |
| 단계별 훈련 가이드 | 민트 bg — 부모가 따라 할 수 있는 4단계 훈련법 (1단계: 바람 느끼기 / 2단계: 뱀 소리 내기 / 3단계: 연결하기 / 4단계: 단어 및 문장), 각 단계에 "포인트" 또는 "가이드" 문구 포함 |
| 유사 패턴 추천 카드 | 라벤더 bg — "ㅅ이 포함된 기초 단어로 집중 연습하세요" + AI 추천 단어 목록 (칩/배지 형태, 예: 사탕, 수박, 사자) |
| 해결책 인포 카드 | 민트 bg — 부모가 아이와 함께 연습할 방법 단계별 안내 |
| 아이연습 시작 버튼 | 피치 버튼 "아이와 연습 시작하기" → 아이연습 페이지로 이동 (약점 단어 + 추천 단어 함께 전달) |
| 기존 오답노트 목록 | 이전에 입력한 단어들 카드 리스트 (날짜 + 오류 음소 배지), 각 카드에 "약점 패턴" 라벨 표시 |

### 6. 아이연습 `/dashboard/practice` — 아이 전용

**목적:** 오답노트 단어 + AI가 추천한 유사 패턴 단어를 게임처럼 반복 연습

**배경:** `135deg 피치→민트→라벤더` 그라디언트 전체화면

| 섹션 | 내용 |
|------|------|
| 단어 목록 | 오답노트 단어(핑크 배지) + AI 추천 유사 패턴 단어(라벤더 배지) |
| 단어 카드 | 이모지(크게) + 단어(font-black) + 음절 분리(· 구분자) |
| 반복 도트 5개 | 회색 → 민트(성공) — 한 단어당 5번 반복 |
| 정답 버튼 | "맞아요! 🌟" (피치, 크게) / "다시" (흰 배경) |
| 정답 시 | confetti 폭발 애니메이션 |
| 단어 완료 시 | 저장하기 버튼 — 누르면 기록에 저장됨 |
| 5회 완료 배너 | 민트 bg "잘했어요! 저장됐어요 🎉" |

### 7. 저장 `/dashboard/saved`

**목적:** 아이연습에서 저장한 단어들과 학습 기록 확인

| 섹션 | 내용 |
|------|------|
| 마스코트 + 레벨 | 5레벨 시스템, 글로우 링 색상 변화 |
| 통계 3열 | 총 단어수 / 정확도 / 연속일 — 색별 카드 |
| 주간 리포트 | 2개 카드: 이번 주 단어수 + 정확도% |
| 잔디 캘린더 | 12주 GitHub식 그리드 — 4단계 색상 (회색→연피치→피치→진한 오렌지), 호버 시 1.3x 확대 + 툴팁 |
| 자주 틀리는 음소 | 핑크 bar 차트 상위 6개 |
| 저장한 단어 목록 | 카드 리스트 형태 — 단어 + 배지(민트/핑크), 오류 음소 `ㅅ→ㄷ` 표시, 날짜 |
| 레벨업 프로그레스 | 피치 진행바 + 격려 메시지 |

## 인터랙션 상태

### 버튼
- **기본:** shadow-bubble
- **Hover:** translateY -1px, shadow-hover
- **Active:** scale 0.96
- **Disabled:** opacity 50%, cursor not-allowed

### 입력 필드
- **기본 보더:** 2px `#F0E8E0`
- **포커스 보더:** 2px `#FFB38A` (피치)
- **플레이스홀더:** `#C4B5A8`

### 카드 선택 (단어/모드 선택)
- **미선택:** 흰 bg, 라이트 보더
- **선택:** 피치/민트 bg, 진한 컬러 보더 2px

## 핵심 플로우

### 부모 플로우
1. 대시보드 홈 → "오답노트 작성하기" 클릭
2. 오답노트 → 옳은 발음 + 오답 입력 → "AI 분석하기"
3. AI 분석 결과 확인 → 해결책 인포 확인
4. "아이와 연습 시작하기" → 아이연습으로 이동

### 아이 플로우
1. 아이연습 페이지에서 오답노트 단어 + AI 추천 단어 확인
2. 단어당 5회 반복 연습 (도트로 진행도 표시)
3. 정답 시 confetti 폭발, "저장하기" 버튼 클릭
4. 저장된 단어는 "저장" 탭에서 확인 가능

## STITCH / Figma 전달 핵심 메모

1. **모바일 단일 레이아웃** — 데스크탑 버전 없음, 최대폭 512px, 세로 스크롤
2. **한국어 전용** — 모든 카피는 한국어, Pretendard 폰트 필수
3. **마시멜로 테마** — 직각/날카로운 모서리 전혀 없음, 최소 radius 12px
4. **색상 규칙:** 피치=주 액션, 민트=성공/보조, 핑크=오류 — 이 3색 규칙 엄수
5. **글래스모피즘** — 헤더/하단 nav만 적용 (blur + 반투명 배경)
6. **애니메이션 필수 구현:** float, bounce-in, dot-pop, confetti
7. **마스코트** — 파랑새 SVG 캐릭터, 자체 blink/wing 애니메이션 내장
8. **레벨 시스템** — 5단계, 각 레벨별 글로우 색상 변화
9. **부모/아이 역할 분리** — 오답노트(부모) ↔ 아이연습(아이) 명확한 구분
10. **5회 반복 메커니즘** — 한 단어당 정확히 5번 도트로 추적, 완료 후 저장
