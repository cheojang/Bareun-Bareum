# lessons.md — 교훈 (한 줄 원칙 + 근거)

> 헤르메스(06)가 관리. 30줄 상한 — 추가하려면 덜 중요한 것을 지운다. 근거 없는 항목 금지.

1. **버그 수정 완료 보고 전, 증상이 났던 경로 그대로 재현 검증한다.** — 문장 소리 잔류는 3회 수정 소요 (근거: PR #136, #137, #143)
2. **스키마/버킷/환경변수 변경은 코드와 별개로 프로덕션 반영 여부를 항상 대조한다.** — 미반영 3연발 (근거: UserConsent, screenshotUrl 컬럼, images 버킷)
3. **오디오 수정은 3계층(재생 중 / 합성 대기 / 폴백 지연) 모두 확인해야 끝난 것이다.** (근거: PR #136→#143 여정)
4. **같은 브랜치에서 추가+삭제 후 스쿼시 머지하면 순변화 0으로 유실된다 — 삭제는 main 병합 후 별도 커밋으로.** (근거: PR #132 유실 → #133 보정)
5. **폴백 경로는 주 경로와 다른 특성(목소리·타이밍)을 가진다 — 폴백 발동 조건(레이트리밋·검증 규칙)을 먼저 의심한다.** (근거: 음절 목소리 뒤바뀜, PR #139)
6. **아동 화면은 "글 못 읽는 아이가 이 과제를 수행 가능한가"를 배포 전에 물어야 한다.** (근거: 그림 없는 변별 게임, PR #140)
7. **프로덕션 장애 조사는 추측 대신 로그(digest)부터 — 코드가 멀쩡해 보여도 인프라(커넥션·도메인)일 수 있다.** (근거: EMAXCONNSESSION, PKCE 도메인 불일치)
8. **머지 후 프로덕션 배포는 반드시 확인한다 — 프리뷰 성공 ≠ 프로덕션 배포됨.** (근거: PR #130 배포 유실 1시간)
