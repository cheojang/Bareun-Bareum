# 업무 스타일 진단 — 설계의 근거

바른발음 개발·운영 세션 전체와 깃허브 레포 11개(2026.03~07)를 분석한 결과.

## 관찰된 패턴

### 1. 스크린샷 지시형 개발
- 요청의 절반 이상이 모바일 스크린샷 첨부 + 짧은 한국어 지시.
- 코드 위치를 지정하지 않음 — 에이전트가 화면에서 컴포넌트를 역추적해야 함.
- → **함의**: 에이전트는 "화면 → 코드 위치" 매핑 능력이 핵심. 보고도 화면 언어(버튼 이름, 화면 이름)로.

### 2. 실사용 QA 감각
- "소리1을 누르면 차과가 나오는데 다시듣기는 사과→차과 순서다" — 재생 순서 불일치 발견
- "글자마다 목소리가 여자였다 남자였다 한다" — 레이트리밋 폴백을 귀로 감지
- → **함의**: 사람의 감각 QA는 대체가 아니라 **증폭** 대상. 에이전트는 그 전 단계(명백한 결함)를 걸러 사람 QA의 밀도를 높인다.

### 3. 완전 위임 + 정책 직접 설계
- 커밋→PR→머지→배포→검증 전 과정을 "진행해" 한마디로 위임.
- 반면 보상 곡선(2번 인증=1개월), 심사 정책(2일 자동승인), 가격(4,900원)은 직접 설계.
- → **함의**: 에이전트는 실행은 끝까지, **정책 결정은 반드시 사람에게**. 옵션 A/B/C 제시 형식이 잘 맞음.

### 4. 다산형 MVP 메이커
- 3개월간 레포 11개: 아동교육(바른발음), 안전(Go_dok), 건강(HealthVibe), 로컬비즈(flower, ROOT_RESTRANT),
  도구(workSpace, CAR-COMPARE-PRICE, 2st_Brain), 재미(DESKTOP_PET, who_is_same).
- 다수 레포의 기본 브랜치가 `claude/...` — Claude Code로 초기 구축.
- → **함의**: "새 아이디어 → 표준 스택 뼈대"의 반복 비용이 큼. 스캐폴더의 가치가 높다.

### 5. 비용 감각
- 로컬 엔진 70% / AI 30% 분담 설계, TTS 영구 캐시, 이미지 생성 크레딧 관리.
- → **함의**: 에이전트 운영도 저비용 우선 — 크론+캐시 우선, LLM 호출은 판단이 필요한 지점에만.
