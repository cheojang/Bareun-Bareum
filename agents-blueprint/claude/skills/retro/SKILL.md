# 세션 회고 (헤르메스 자가개선 루프)

이번 세션(대화 맥락 전체)을 되돌아보고 아래 절차로 지식을 축적한다.

## 절차

1. **반복 수정 건 추출** — 같은 증상을 2회 이상 고친 사례를 찾는다.
   각각에 대해: 왜 첫 수정이 빗나갔나? 어떤 검증을 했다면 1회차에 잡았나?

2. **새 실패 유형 판별** — `agents-blueprint/knowledge/patterns.md`에 없는 유형이면
   `증상 → 원인 → 검증법` 형식으로 추가를 제안한다.

3. **교훈 추출** — 다음 세션에 일반화 가능한 한 줄 원칙만.
   `agents-blueprint/knowledge/lessons.md`에 추가 제안. **반드시 근거(커밋/PR)를 병기**한다.
   근거 없는 일반화는 금지. 파일이 30줄을 넘으면 덜 중요한 항목 삭제를 함께 제안.

4. **에이전트 정의 개선 제안** — `.claude/agents/*.md`, `.claude/skills/*/SKILL.md` 중
   이번 세션에서 부족했던 지식·체크리스트가 드러난 것이 있으면 구체적 diff로 제안한다.

5. **모델 라우팅 검토** — 하위 모델이 반복 실패했거나 상위 모델이 기계적 일에 쓰인 사례가 있으면
   `agents-blueprint/agents/07-model-router/README.md`의 분배표 갱신을 제안한다.

## 출력 형식
```
📋 세션 회고
· 반복 수정: N건 (요약 + 1회차에 잡을 수 있었던 검증)
· patterns.md 추가 제안: ...
· lessons.md 추가 제안: ... (근거: PR #N)
· 정의 개선 제안: kid-ux-reviewer.md에 "..." 추가 (diff 제시)
→ 반영할까요? (지식 파일은 승인 시 직접 수정, 정의 파일은 PR로)
```

## 금지
- 승인 없이 에이전트 정의 파일 수정 금지 (지식 파일은 승인 후 직접 수정 가능)
- 사례 1건으로 일반 원칙 만들기 금지 — 2회 이상 반복되었거나 비용이 컸던 것만
