# 바른발음 AI 분석 로직 명세서

## 개요

부모가 **[목표 발음 / 아이의 발음]** 을 입력하면, AI가 다음 3단계로 응답한다.

```
1. 진단     → 어떤 오류인지 전문 용어 + 쉬운 설명
2. 원인 분석 → 왜 그 소리가 나는지 조음 원리 설명
3. 훈련 가이드 → 단계별 실전 훈련법 + 추천 단어
```

---

## 시스템 아키텍처

### 3단계 하이브리드 분석 엔진

```
[1단계] 로컬 자모 분리 엔진 (코드, 무료) — 약 70% 커버
  └─ 한글 유니코드 분해: 초성/중성/종성 추출
  └─ 대치/탈락/첨가 패턴 자동 판정
  └─ 동화 오류는 "미판정" 표시 후 2단계로

[2단계] Gemini Flash API (저렴, 빠름)
  └─ 동화/복합 오류/설소대 증상 최종 판정 (30%)
  └─ 원인 설명(쉬운 말), 4단계 훈련법, 추천 단어 생성
  └─ 누적 약점 분석 리포트 생성

[3단계] DB 집계 (코드, 무료)
  └─ 음소별 오류율 계산
  └─ 약점 판정 (30% / 50% 기준)
```

### 로컬 엔진 가능 범위

| 카테고리 | 로컬 | 방법 |
|---------|------|------|
| 대치 (7개) | ✅ | 자모 1:1 비교 + 패턴 테이블 |
| 탈락 (4개) | ✅ | 음절 수 + 자모 수 비교 |
| 첨가 (3개) | ✅ | 자모 수 비교 |
| 동화 (4개) | ⚠️ | 앞뒤 맥락 필요 → Gemini |

## 조음 오류 전수 리스트 (20개 패턴)

아동 조음 오류를 언어치료학 관점에서 4대 카테고리로 분류한 진단 기준 데이터입니다.

## 1. 대치 (Substitution): 7개 패턴

소리를 다른 소리로 바꾸는 경우. 가장 빈번한 오류.

| # | 패턴명 | 예시 | 원인 |
|---|--------|------|------|
| 1 | 경음화 (Tensing) | 사과 → 따과 | 예사소리를 된소리로 강하게 발음 |
| 2 | 기음화 (Aspiration) | 구두 → 쿠두 | 예사소리를 거센소리로 터뜨려 발음 |
| 3 | 치조음화 (Fronting) | 가방 → 다방 | 입 뒤 소리를 입 앞 소리로 당겨 발음 |
| 4 | 연구개음화 (Backing) | 바지 → 가지 | 입 앞 소리를 목구멍 쪽 소리로 밀어 발음 |
| 5 | 마찰음 파열음화 | 사자 → 다자 | 공기를 흘려보내지 못하고 입천장을 터뜨림 |
| 6 | 유음의 비음화 | 라디오 → 나디오 | 혀 옆 흐름(ㄹ)을 코울림(ㄴ)으로 발음 |
| 7 | 활음화 (Gliding) | 오이 → 웨이 | 단모음을 이중모음으로 바꾸어 발음 |

### 패턴 1-1. 경음화 (Tensing)
- **예시:** 사과 → **따**과
- **원인:** 혀가 입천장을 '쾅' 닫아서 강한 소리 발생
- **훈련:** "바람 느끼기 → 뱀 소리 내기 → 음절 연결 → 단어 확장"
- **추천 단어:** 사탕, 사자, 사다리, 수박, 소나무

### 패턴 1-2. 기음화 (Aspiration)
- **예시:** 구두 → **쿠**두
- **원인:** 거센소리와 예사소리의 구분이 어려움
- **훈련:** "숨 강약 조절 → 거센 vs 부드러운 비교 → 단어로 연습"
- **추천 단어:** 구두, 기차, 고양이, 국수, 과자

### 패턴 1-3. 치조음화 (Fronting)
- **예시:** 가방 → **다**방
- **원인:** 혀 뒤쪽(연구개) 조절이 미숙해 혀 앞쪽(치조) 사용
- **훈련:** "목구멍 깊이 감각 → 기침 소리 연결 → 음절 연결 → 단어"
- **추천 단어:** 가방, 기차, 고양이, 거북이, 구름

### 패턴 1-4. 연구개음화 (Backing)
- **예시:** 바지 → **가**지
- **원인:** 혀 뒤쪽으로 소리를 밀어서 발음
- **훈련:** "입 앞 감각 → 치아 뒤 위치 연습 → 음절 → 단어"
- **추천 단어:** 바지, 박수, 버스, 배, 볼

### 패턴 1-5. 마찰음 파열음화
- **예시:** 포도 → **뽀**도 / 사자 → **다**자
- **원인:** 공기 흐름 조절 실패로 입천장을 터뜨림
- **훈련:** (경음화 참고)
- **추천 단어:** 포도, 사자, 씨앗, 소리, 신발

### 패턴 1-6. 유음의 비음화
- **예시:** 라디오 → **나**디오
- **원인:** ㄹ은 혀를 튕기는 동작이 어려워 ㄴ으로 대체
- **훈련:** "혀 튕기기 → 라라라 반복 → 음절 연결 → 단어"
- **추천 단어:** 라디오, 라면, 로봇, 나라, 사랑

### 패턴 1-7. 활음화 (Gliding)
- **예시:** 오이 → **웨**이
- **원인:** 단모음을 이중모음으로 변형
- **훈련:** "입 모양 고정 → 모음만 발음 → 단어 연습"
- **추천 단어:** 오이, 아이, 외출, 우유, 올바른

## 2. 동화 (Assimilation): 4개 패턴

주변 소리를 닮아가는 경우. 입 근육 전환이 빠르지 않을 때 발생.
**주의:** 로컬 엔진으로는 판정 어려움 → Gemini Flash로 위임

| # | 패턴명 | 예시 | 원인 |
|---|--------|------|------|
| 8 | 역행 동화 | 손님 → 솜님 | 뒤의 ㄴ을 준비하려다 앞 받침이 변함 |
| 9 | 순행 동화 | 종로 → 종노 | 앞 글자의 영향으로 뒤 소리가 변함 |
| 10 | 양순 동화 | 신발 → 심발 | 뒤의 입술소리(ㅂ) 때문에 앞 받침이 ㅁ으로 변함 |
| 11 | 연구개 동화 | 건강 → 겅강 | 뒤의 목구멍소리(ㄱ) 때문에 앞 받침이 변함 |

## 3. 탈락 (Omission): 4개 패턴

특정 소리를 빼먹는 경우. 명료도를 가장 크게 떨어뜨리는 요인.

| # | 패턴명 | 예시 | 원인 |
|---|--------|------|------|
| 12 | 초성 탈락 | 사자 → 아자 | 단어의 첫 자음을 발음하지 않음 |
| 13 | 종성(받침) 탈락 | 수박 → 수바 | 받침 소리를 내지 않고 모음으로 끝냄 |
| 14 | 유음 탈락 | 하늘 → 하느 | ㄹ 받침을 생략함 |
| 15 | 음절 축약 | 고양이 → 괭이 | 여러 음절을 하나로 합쳐 발음 |

### 패턴 3-1. 초성 탈락
- **예시:** 사자 → **아**자
- **원인:** 음절 구조 단순화 (C+V 선호)
- **훈련:** "초성 강조하기 → 음절 분리 → 정상 속도"
- **추천 단어:** 사자, 사과, 소리, 수박, 손

### 패턴 3-2. 종성(받침) 탈락
- **예시:** 수박 → 수**바**
- **원인:** 받침이 있다는 것을 인식하지 못함
- **훈련:** "입 닫기 감각 → 받침 강조 → 정상 속도"
- **추천 단어:** 수박, 밥, 컵, 공, 방

### 패턴 3-3. 유음 탈락
- **예시:** 하늘 → 하**느**
- **원인:** ㄹ 발음의 복잡성으로 생략
- **훈련:** "혀 튕기기 → 라라라 → 받침 ㄹ 강조"
- **추천 단어:** 하늘, 라면, 나라, 달, 말

### 패턴 3-4. 음절 축약
- **예시:** 고양이 → **괭**이
- **원인:** 음절을 정확히 나누지 못함
- **훈련:** "음절 분리 → 손뼉 치며 발음 → 정상 속도"
- **추천 단어:** 고양이, 나비, 도마뱀, 거미, 나팔

## 4. 첨가 및 기타 (Addition & Etc): 3개 패턴

없는 소리를 넣거나 공기 흐름이 잘못된 경우.

| # | 패턴명 | 예시 | 원인 |
|---|--------|------|------|
| 16 | ㄴ 첨가 | 솜사탕 → 솜**난**탕 | 단어 사이에 ㄴ을 삽입함 |
| 17 | 비음화 (Nasalization) | 학교 → **항**교 | 입으로 나갈 바람이 코로 새어 콧소리 됨 |
| 18 | 설소대 증상 | 라면 → 다면 | 혀 아래 인대가 짧아 혀가 천장에 닿지 못함 |

## 누적 데이터 약점 분석 로직

### 집계 방식

부모가 입력한 오답 데이터를 누적해서 **음소별 오류 빈도**를 계산한다.

```
오류 패턴 빈도 계산:
- 최근 300개 오답 기록 기준 (약 5개월 데이터)
- 음소별 오류 횟수 / 전체 오답 수 × 100 = 오류율(%)
- 오류율 30% 이상인 음소 → "약점 음소"로 분류
```

**선택 이유 (300개가 최적):**
- 통계 신뢰도: ±5.7% 오차율 (±9.8%보다 훨씬 정확)
- 시간 범위: 5개월 = 현실적인 장기 패턴 반영
- 음소 데이터: 각 음소당 45~60회 이상 (통계적 충분성)
- 부모 피드백: 명확한 개선/악화 추이 감지 가능
- 성능: DB 용량 30MB, 쿼리 0.022ms (무시할 수준)

**추가 규칙:**
- 100개 미만: "아직 데이터가 부족해요. 더 연습하면 더 정확한 분석을 드릴 수 있어요!"
- 100~300개: 신뢰도 점진적 증가 표시
- 300개 이상: "최근 5개월간의 신뢰도 높은 분석입니다"

### 약점 판정 기준

| 오류율 | 등급 | 표시 |
|--------|------|------|
| 50% 이상 | 집중 교정 필요 | 핑크 배지 (진함) |
| 30~49% | 꾸준한 연습 필요 | 핑크 배지 (연함) |
| 10~29% | 관찰 중 | 라벤더 배지 |
| 10% 미만 | 정상 범위 | 민트 배지 |

### AI 진단 메시지 포맷

```
[진단 카드 텍스트 예시]

최근 20번의 연습에서 'ㅅ' 소리를
17번 틀렸어요 (오류율 85%)

이 소리는 "마찰음"으로,
혀와 입천장 사이로 공기를
'스으~' 흘려보내야 해요.

지금 아이에게 가장 필요한 연습이에요!
```

## Gemini Flash API 설정

### 시스템 프롬프트

```
당신은 한국어 아동 조음 교정 전문가입니다.
부모가 이해할 수 있는 쉬운 말로 설명해주세요.
항상 희망적이고 격려하는 톤으로 작성하세요.
```

### 유저 프롬프트 템플릿

```
목표 단어: {target_word}
아이 발음: {child_word}
판정된 오류 유형: {error_type} ({error_category})
아이 나이: {child_age}개월
최근 약점 음소: {weak_phonemes}

다음을 한국어로 출력하세요:

1. 원인 분석 (2~3문장, 쉬운 말)
   - 조음 원리를 6세 부모도 이해할 수 있게

2. 단계별 훈련법 4단계
   - 각 단계: 제목, 방법, 포인트/가이드

3. 추천 연습 단어 5개
   - 쉬운 단어부터 난이도 증가

4. 부모 응원 메시지 (1문장)
```

### API 호출 최적화

- **로컬로 판정 가능 (70%):** API 호출 안 함
- **미판정 오류 (30%):** API 호출 (동화, 복합 오류)
- **비용 예상:** 월 1,000건 기준 약 $2 (Claude API 대비 1/10)

```python
    음소별 오류율 계산
    """
    errors = db.query(f"SELECT * FROM errors WHERE user_id={user_id} ORDER BY created DESC LIMIT {recent_count}")
    frequency = {}
    for error in errors:
        frequency[error.error_phoneme] = frequency.get(...) + 1
    
    # 오류율 > 30% → 약점 판정
    weak_phonemes = {k: v/len(errors)*100 for k,v in frequency.items() if v/len(errors) > 0.3}
    return weak_phonemes
```
